“合并”另一分支的部分文件

Posted on 2018-05-31

场景描述

假设现在需要先把dev中的a.cpp合并到master，再继续dev的开发。

解决方法

因为dev分支中对a.cpp做修改的commit都包含有其他的文件，所以这种情况下使用Merge或Rebase都会导致引入不需要的文件。而使用checkout则可以完成此操作；

git checkout dev a.txt 实际上是用另一个分支的文件覆盖了当前分支的文件，并不属于合并操作。
