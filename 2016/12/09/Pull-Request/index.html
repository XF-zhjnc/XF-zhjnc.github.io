Fork A Repo

将开源项目复制到自己的仓库并pull到本地

Set upstream

为了保证fork的项目能跟进上游项目的变更，必须设置上游项目的仓库地址

$ git remote add upstream https://github.com/User/project

设置好后检查本地关联的远程仓库：

$ git remote -v

origin    https://github.com/XF-zhjnc/project.git (fetch)
origin    https://github.com/XF-zhjnc/project.git (push)
upstream    https://github.com/User/project (fetch)
upstream    https://github.com/User/project (push)

其中：

1.origin 是自己GitHub上fork出来的仓库
2.upstream 是上游项目的仓库

Sync upstream

当上游项目有了更新，可以使用fetch指令将上游更新取回本地

$ git fetch upstream
//也可以只取某个特定的分支的更新，具体参数加在后面

上游更新取回本地后，查看一下本地当前的所有分支情况

$ git branch -a

若想与上游项目某个分支代码合并，先切换到想合并的本地分支，再使用merge合并

$ git checkout 本地分支名    
$ git merge upstream/分支名

另外本地的代码更新和分支更新应及时push到GitHub

Create your topic branch

一般情况下不应该直接在默认分支master上对项目进行修改，应该为每个工作创建一个单独的分支

e.g. 修复项目中的一个issue

$ git checkout -b fix-Permission-err

push新分支信息到GitHub并开始在该分支下工作

Rebase & Merge

  • 如果在主题分支中工作的过程中，想要导入来自上游库(与当前的工作不冲突的情况下)的更新

    $ git rebase upstream/master 本地主题分支名<如果当前分支就是则可以省略>

这将把当前branch的开发“base（基础）”推进到一个新的起点，而不会引入多余的commits。

  • 如果在主题分支中工作的过程中，想要合并别的分支

    $ get merge 另一个分支名
    • 如果merge的分支来自远程库，一次merge操作会增加一个额外的commit操作（“Merge branch ‘master’ of something”）
    • git merge还可以用来将其它主题分支引入的新功能合并到主线分支，以更新项目开发进度

Pull Request

在GitHub上fork出来的仓库首页，New pull request，选择base branch以及head branch

  1. base branch：希望pull request被合并到上游项目的分支，也就是目标分支。
  2. head branch：想要提交的自己开发库中的主题分支。

如果在某个分支里推送了一条pull request，那么在这条pull request被关闭之前，所有commit和push到该分支的代码都会追加到这个pull request后面

Delete Branch

某个主题分支的使命完成后，删除掉该分支

$ git branch -d fix-Permission-err
